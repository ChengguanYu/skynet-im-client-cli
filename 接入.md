# C# KCP 客户端接入指南

本文档指导如何用 C# 编写 KCP 客户端，连接到本服务的 `skynet + lkcp` KCP 服务器。

服务器监听 **本地 UDP 8888 端口**，协议为 **裸 KCP over UDP**（无任何外层封装），收到完整上层消息后回射 `echo + recv_data`。

---

## 1. 协议概览

| 项目 | 值 |
|---|---|
| 传输层 | UDP |
| 应用层 | KCP（可靠传输） |
| 服务器地址 | `服务器IP:8888` |
| 线材封装 | **裸 KCP segment**，无外层 mux、无长度前缀 |
| 字节序 | **小端（Little-Endian）**，所有多字节字段 |
| 回包格式 | 同样是裸 KCP segment，内容为 `echo` + 收到的数据 |

**重要**：本服务**没有握手过程**。客户端发来的第一个 UDP 包只要 conv 合法、是合法 KCP segment，服务端就会立即建立会话。conv 由**客户端单方面决定**，服务端被动接受，不校验唯一性（按 conv 路由会话表）。

---

## 2. KCP Segment 线材格式

每个 UDP 包 = 一个或多个 KCP segment。segment 头 24 字节，结构如下（来自 `ikcp.c` `ikcp_encode_seg`）：

| 偏移 | 长度 | 字段 | 类型 | 字节序 | 说明 |
|---|---|---|---|---|---|
| 0 | 4 | `conv` | uint32 | LE | 会话 ID，客户端自定，服务端据此路由 |
| 4 | 1 | `cmd` | uint8 | - | 命令：81=PUSH, 82=ACK, 83=WASK, 84=WINS |
| 5 | 1 | `frg` | uint8 | - | 分片序号（0 = 末片或未分片） |
| 6 | 2 | `wnd` | uint16 | LE | 接收窗口大小（可发送包数） |
| 8 | 4 | `ts` | uint32 | LE | 时间戳（毫秒） |
| 12 | 4 | `sn` | uint32 | LE | 序列号 |
| 16 | 4 | `una` | uint32 | LE | 待确认序列号（此 sn 之前的都已收到） |
| 20 | 4 | `len` | uint32 | LE | 负载长度（字节） |
| 24 | N | `data` | bytes | - | 负载（N = len） |

**关键约束**：
- 头长度固定 24 字节（`IKCP_OVERHEAD`）
- 默认 MTU = 1400 字节（含头），**单段负载上限 MSS = 1376 字节**
- 单个 UDP 包可含多个 segment（KCP 会合并发送），总长 ≤ `(MTU + OVERHEAD) * 3 = 4272` 字节
- **客户端只需调 `Send(data)`，分片/打包由 KCP 库自动完成**，不用手动构造上面的头

## 3. 服务端行为详解

### 3.1 收包流程（`src/main.lua`）

```
UDP 包到达
  ↓
on_udp_data(str, address)
  ↓
read_conv(str)        ← string.unpack("<I4", str) 从偏移0读 conv(小端)
  ↓
get_session(conv, address)
  ├─ sessions[conv] 已存在 → 复用
  └─ 不存在 → lkcp.lkcp_create(conv, output) 建新会话, 记日志
  ↓
session.address = address   ← 记录对端地址(NAT变化时更新)
  ↓
kcp:lkcp_input(str)         ← 整包送入 KCP 协议栈
  ↓ (KCP 内部重组、去重、确认)
循环 kcp:lkcp_recv()        ← 取出已完整的上层消息
  ↓
handle_message(session, msg):
  - 打印 [KCP] recv conv=0x.. len=.. data=".."
  - kcp:lkcp_send("echo" .. msg)   ← 回射 echo + 原数据
```

### 3.2 发包路径

KCP 在 `lkcp_update` / `lkcp_flush` 时触发 output 回调：

```lua
function(buf)
    socket.sendto(udp_id, session.address, buf)   -- 直接发裸 KCP segment
end
```

**回包就是裸 KCP segment**，客户端 `kcp.Input()` 喂进去即可。

### 3.3 KCP 参数（服务端配置）

服务端建会话时设置（`src/main.lua` 第 56-57 行）：

```lua
kcp:lkcp_nodelay(1, 10, 2, 1)    -- 快速模式
kcp:lkcp_wndsize(128, 128)       -- 窗口 128/128
```

| 参数 | 值 | 含义 |
|---|---|---|
| nodelay | 1 | 启用快速模式 |
| interval | 10 ms | KCP 内部 update 间隔 |
| resend | 2 | 快速重传指标（2 倍 ACK 触发重传） |
| nc | 1 | 关闭拥塞控制 |
| sndwnd | 128 | 发送窗口 |
| rcvwnd | 128 | 接收窗口 |
| mtu | 1400 | 默认值（未改） |

**客户端应使用相同参数**，否则行为不对称、性能下降。

### 3.4 update 定时器

服务端每 10 ms 对所有会话调一次 `lkcp_update`。客户端**必须**也定期调 `Update`（建议 10 ms），否则 KCP 状态机不推进，数据发不出、收不到。

## 4. C# 库选型

### 推荐：`kcp2k`

| 项 | 值 |
|---|---|
| NuGet 包 | `kcp2k` |
| 安装 | `dotnet add package kcp2k` |
| Namespace | `kcp2k` |
| 仓库 | https://github.com/MirrorNetworking/kcp2k |
| 特点 | kcp.c v1.7 的 line-by-line C# 翻译，线材 100% 兼容 |
| 生产验证 | Mirror Networking 在用（大量 Unity 游戏） |

**为什么选 kcp2k**：线材兼容性最稳——逐行翻译 C 版 kcp，`Encode32U` 就是 `p[0]=low, p[3]=high`（小端），与服务端 `ikcp_encode32u` 完全一致。

**只用低层 `Kcp` 类**，不要用 `KcpClient` / `KcpPeer`（高层加了 channel byte 和握手协议，与本服务不兼容）。

### 备选：`KumoKyaku/KCP`

| 项 | 值 |
|---|---|
| NuGet 包 | `Kcp` |
| Namespace | `System.Net.Sockets.Kcp` |
| 特点 | Span-based API，线程安全，零分配，ET 框架在用 |

注意：它有 `IsLittleEndian` 开关，**默认 true（正确）**，不要改。

### 不要用

- `Cysharp/KcpTransport`：自带 Syn Cookie 握手和额外 framing，**不兼容**裸 KCP 服务端。

## 5. kcp2k 核心 API

| 方法 | 签名 | 说明 |
|---|---|---|
| 构造 | `new Kcp(uint conv, Action<byte[], int> output)` | 创建 KCP。`conv` 为会话 ID；`output` 回调在 KCP 要发包时被调 |
| `Input` | `int Input(byte[] data, int offset, int size)` | 把收到的 UDP 裸包喂进 KCP。返回 0 成功，负数出错 |
| `Send` | `int Send(byte[] buf, int offset, int len)` | 提交上层待发数据。返回 0 成功 |
| `Receive` | `int Receive(byte[] buf, int len)` | 读取已完整的上层消息。返回读到的字节数，负数表示无数据 |
| `PeekSize` | `int PeekSize()` | 查看下一条消息长度（不消费） |
| `Update` | `void Update(uint currentMs)` | 驱动状态机。每 10 ms 调一次 |
| `Check` | `uint Check(uint currentMs)` | 返回下次该调 Update 的时间戳（优化用） |
| `SetNoDelay` | `void SetNoDelay(uint nodelay, uint interval, int resend, bool nocwnd)` | 配置快速模式 |
| `SetWindowSize` | `void SetWindowSize(uint sndwnd, uint rcvwnd)` | 配置窗口 |
| `SetMtu` | `void SetMtu(uint mtu)` | 配置 MTU（默认 1200，**建议设 1400 与服务端一致**） |
| `Flush` | `void Flush()` | 立即冲刷待发数据 |

## 6. 完整可运行示例

### 6.1 项目结构

```
KcpEchoClient/

└── Program.cs
```

### 6.2 csproj


> 实际版本号用 `dotnet add package kcp2k` 时拉到的最新版替换。

### 6.3 Program.cs

```csharp
using System.Net.Sockets;
using System.Text;
using kcp2k;

// ===== 配置 =====
const string SERVER_HOST = "127.0.0.1";   // 服务器 IP（远程改成实际地址）
const int    SERVER_PORT = 8888;
const uint   CONV        = 0x11223344;    // 会话 ID, 客户端自定, 服务端被动接受

// ===== 建立 UDP =====
using var udp = new UdpClient();
udp.Connect(SERVER_HOST, SERVER_PORT);

// ===== 创建 KCP =====
// output 回调: KCP 要发包时, 通过 UDP 发出去
var kcp = new Kcp(CONV, (data, size) =>
{
    udp.Send(data, size);
});

// 配置与服务端一致
kcp.SetNoDelay(1, 10, 2, true);   // nodelay=1, interval=10ms, resend=2, nocwnd=true
kcp.SetWindowSize(128, 128);
kcp.SetMtu(1400);

// ===== 收包循环: UDP -> KCP.Input =====
var recvBuf = new byte[4096];
_ = Task.Run(async () =>
{
    while (true)
    {
        var res = await udp.ReceiveAsync();
        // 把 UDP 裸包喂进 KCP 协议栈
        kcp.Input(res.Buffer, 0, res.Buffer.Length);
    }
});

// ===== Update + 应用层收数循环 =====
_ = Task.Run(async () =>
{
    var appBuf = new byte[65536];
    while (true)
    {
        uint now = (uint)Environment.TickCount;
        kcp.Update(now);

        // 尝试取出完整的应用层消息
        int n = kcp.Receive(appBuf, appBuf.Length);
        if (n > 0)
        {
            string msg = Encoding.UTF8.GetString(appBuf, 0, n);
            Console.WriteLine($"[RECV] {n} bytes: {msg}");
        }

        await Task.Delay(10);   // 10ms
    }
});

// ===== 主线程: 发送测试 =====
Console.WriteLine("输入要发送的内容, 回车发送 (空行退出):");
while (true)
{
    string? line = Console.ReadLine();
    if (string.IsNullOrEmpty(line)) break;
    byte[] data = Encoding.UTF8.GetBytes(line);
    kcp.Send(data, 0, data.Length);
    // 可选: kcp.Flush() 立即冲刷, 否则等下次 Update 时发
}

kcp.Flush();
```

### 6.4 运行

```bash
dotnet run
```

输入 `hello` 回车，预期：
- 服务器日志：`[KCP] new session conv=0x11223344 addr=<你的IP>` + `[KCP] recv conv=0x11223344 len=5 data="hello"`
- 客户端输出：`[RECV] 9 bytes: echohello`

## 8. 关键注意事项

### 8.1 字节序
KCP 线材格式**永远是小端**。kcp2k 默认小端，无需手动处理。**不要**尝试改大端，否则 `ikcp_input` 会因 `conv != kcp->conv` 拒包。

### 8.2 conv 选择
- 客户端单方面决定，服务端被动接受
- 同一客户端断线重连建议用相同 conv（服务端按 conv 复用会话）—— 但前提是旧会话还活着；服务端目前**不清理超时会话**，长时间断线后重连可能命中残留会话，地址会更新但 KCP 内部序号状态可能不一致。生产环境建议重连换新 conv。
- 多个客户端用**不同 conv**，否则服务端会话表撞键、互相串数据

### 8.3 Update 必须定时调
KCP 是主动协议，不调 `Update` 就不发 ACK、不重传、不推进。**至少每 100ms 一次**，低延迟场景 10ms。可用 `Check()` 优化空转。

### 8.4 NAT 与地址更新
服务端每次收包都更新 `session.address`，所以客户端 NAT 地址变化不影响。客户端用 `udp.Connect` 固定目标，不感知服务端地址变化（本服务单机部署，服务端地址不变）。

### 8.5 不要用高层封装
kcp2k 的 `KcpClient`/`KcpPeer`、Cysharp 的 `KcpTransport` 都加了额外握手/framing，**与本服务不兼容**。只用裸 `Kcp` 类。

### 8.6 分片
单次 `Send` 数据 > MSS（1376 字节）时，KCP 自动分片（`frg` 字段递减，0 表末片）。接收端 `Receive` 自动拼回完整消息。客户端无感知。

### 8.7 线程安全
kcp2k 的 `Kcp` 类**不是线程安全**的。`Input`、`Update`、`Send`、`Receive` 不要并发调。示例里用两个 `Task.Run` 并发跑 `Input` 和 `Update`——**生产环境要加锁**或都放到同一线程的队列里串行执行。简化做法：

```csharp
private readonly object _kcpLock = new();
// 所有 kcp 调用都 lock(_kcpLock) { ... }
```

## 9. 调试清单

### 客户端发数据后服务端没反应

1. **确认 UDP 通到服务器**：`nc -u 服务器IP 8888` 敲字符回车，服务端应打印 `[KCP] malformed udp packet, drop` 或 `[KCP] new session ...`。什么都没打印 = UDP 没通（防火墙/地址错）
2. **确认 conv 字节序**：抓包看 UDP payload 前 4 字节。conv=0x11223344 时应为 `44 33 22 11`（小端）。如果是 `11 22 33 44` 就是大端，错了
3. **确认调了 Update**：KCP 握手前的包也走 KCP 状态机，不 Update 发不出
4. **确认 output 回调真的发了**：在回调里加日志，确认 `udp.Send` 被调到

### 服务端建了会话但收不到 echo

1. 看 `lkcp_input` 返回值：`-1` = conv 不匹配（字节序问题）；`-2` = 长度异常；`-3` = cmd 非法
2. 看 `lkcp_recv` 是否返回 0：KCP 需要收齐完整消息（可能多包）才能 `recv` 出来，第一个包可能只是部分数据，等后续包
3. 确认客户端 `Update` 在跑：服务端的 ACK 包要被客户端 `Input` 进去 + `Update` 处理，链路才双向通

### 回射数据客户端收不到

1. 确认服务端 output 回调发了：服务端日志没 `lkcp_send error` 就发出去了
2. 确认客户端 `Input` 在跑：服务端回的包要被客户端 UDP 收到并喂进 `kcp.Input`
3. 确认客户端 `Receive` 在跑：`Update` 后立即 `Receive`，不要只 Update 不 Receive

### 抓包确认线材

```bash
# 在服务器上
sudo tcpdump -i any -X udp port 8888
```
看包前 24 字节是否符合 segment 头格式，conv 字段是否小端。

## 10. 服务端代码参考

完整服务端实现见 `src/main.lua`。关键摘要：

```lua
-- 收包回调
local function on_udp_data(str, address)
    local conv = read_conv(str)              -- string.unpack("<I4", str) 小端读 conv
    local session = get_session(conv, address)  -- 建或查会话
    session.address = address
    session.kcp:lkcp_input(str)              -- 喂入 KCP
    while true do
        local rlen, msg = session.kcp:lkcp_recv()
        if rlen <= 0 then break end
        handle_message(session, msg)         -- 打印 + lkcp_send("echo" .. msg)
    end
end

-- 会话创建
local function get_session(conv, address)
    -- ...
    local kcp = lkcp.lkcp_create(conv, function(buf)
        socket.sendto(udp_id, session.address, buf)   -- 回包直接发裸 segment
    end)
    kcp:lkcp_nodelay(1, 10, 2, 1)
    kcp:lkcp_wndsize(128, 128)
    -- ...
end
```
